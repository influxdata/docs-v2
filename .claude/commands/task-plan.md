# Plan Management (Deprecated)

> **⚠️ Deprecated**: The PLAN.md convention has been replaced by **execution plans**
> under `docs/exec-plans/`. See the [exec-plan-writing skill](../skills/exec-plan-writing/SKILL.md).
>
> For in-session planning, keep notes in your conversation or session memory — don't
> commit ephemeral working notes to the repository.

## Migration

Instead of creating PLAN.md:

1. **For durable decision records** → Create an exec-plan in `docs/exec-plans/active/YYYY-MM-DD-slug.md`
2. **For working notes** → Keep them in-session (conversation history, session memory)
3. **For task tracking** → Use GitHub Issues or PR checklists

## Legacy Documentation

The original `/task-plan` command managed ephemeral PLAN.md files. This convention
has been retired. The automated cleanup workflow has been removed, and PLAN.md
files should no longer be created.

For durable planning documents, use execution plans instead.
